# Report Generation: Guiding Principles

## When to Generate

After any analysis is complete, offer to write a summary. Don't force it.

## What a Good Report Contains

1. **Sample info**: What was imaged, with what microscope, what settings
2. **Methods**: What analysis was done, what parameters, what software
3. **Results**: Key numbers, with proper units and error bars/ranges
4. **Figures**: Clean, publication-quality, with proper axis labels
5. **Interpretation**: What the data means in the context of the user's research question
6. **Caveats**: What could be wrong, what assumptions were made

## Figure Standards

- 200+ dpi for publication
- Proper axis labels with units (µm, µm³, not pixels)
- Scale bars on images
- Colorbar if using pseudocolor
- Font size readable when printed at column width

## Do NOT

- Generate boilerplate text that says nothing
- Over-interpret the data — if you're not sure what it means, say so
- Forget to mention limitations
- Assume you know the scientific context better than the researcher — always check with them
